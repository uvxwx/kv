Этот проект реализует транзакционное хранилище пар ключ-значение, работающее в памяти одного процесса и ориентированное на строгую корректность при конкурентном доступе: в нем есть сериализуемые транзакции с обратным откатом при ошибках, блокировки на уровне отдельных ключей, повторные попытки при конфликте, а также набор многопоточных тестов, проверяющих поведение при чтении, записи, удалении и состязании между параллельными операциями.
